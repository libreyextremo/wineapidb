WINE API DB Documentation

Our API requires moderate technical knowledge, but allows you greater flexibility to customize the search and presentation of our data on your site or app.

Description

Our API is a powerful way of plugging Wine API DB data directly into your website or application. It may be used by wine apps, websites, blogs, market research companies and those specializing in wine selling. The API is a RESTful web service. Queries are passed to it via URLs constructed and submitted using the HTTP protocol (specifically the GET method), just like a web page. Results can be returned in either XML or JSON format.

Url API

The url of the API is http://www.wineapidb.com/api/
Example: Getting all the wines from the Finca Winery
www.wineapidb.com/api/wines

  • URL Encoding & Prohibited Characters

    Parameters

    - Latin1 (ISO-8859-1) is the standard encoding used for our API.
    - 8-bit characters must be encoded with percent encoding ('URL escaping').
    - Percent encoding must be used for all special characters (e.g. â, é, í, ò, ü).
    - Percent and slash symbols used in any way other than percent encoding will cause requests to fail.
    - Parameters are case-sensitive.

    Parameters are case-sensitive, and are passed as querystrings to our base API URL.
    - wineid: This parameter filters by the name of the wine.
    - do: This parameter filters by the denomination of origen of the wine.
    - winery: This parameter filters by the winery of the wine.
